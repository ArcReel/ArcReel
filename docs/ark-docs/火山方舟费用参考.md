# Video Generation Model (Seedance) Billing Description

The video generation model uses **asynchronous inference** and is billed by the number of tokens consumed by the output video (CNY/million tokens).

## 1. Per-Token Pricing Table



| **Model Name**                   | **Online Inference (CNY/million tokens)**           | **Offline Inference (CNY/million tokens)** |
| -------------------------------- | -------------------------------------------------- | --------------------------- |
| **doubao-seedance-2.0**          | - Input without video: 46.00 - Input with video: 28.00 | Not supported              |
| **doubao-seedance-2.0-fast**     | - Input without video: 37.00 - Input with video: 22.00 | Not supported              |
| **doubao-seedance-1.5-pro**      | - With audio: 16.00 - Without audio: 8.00          | - With audio: 8.00 - Without audio: 4.00 |
| **doubao-seedance-1.0-pro**      | 15.00                                              | 7.50                        |
| **doubao-seedance-1.0-pro-fast** | 4.20                                               | 2.10                        |
| **doubao-seedance-1-0-lite**     | 10.00                                              | 5.00                        |

### Billing Rules and Token Estimation Notes:

- **Billing Condition**: Only successfully generated videos are billed. Videos that fail to generate due to review or other reasons are not charged.
- **Video Price Estimation Formula**: `Per-token price × token usage`
- **Seedance 2.0 / 2.0-fast token estimation**: `(input video duration + output video duration) × width × height × fps / 1024`
  - *Note: When input includes video, a minimum token usage applies (see table below); if the estimated usage < minimum, billing is based on the minimum.*
- **Seedance 1.x token estimation**: `(width × height × fps × duration) / 1024`
- **Draft video (480p only) estimation formula**: `normal video token usage × conversion factor`
  - *Note: The token conversion factor for `doubao-seedance-1.5-pro` is: without audio 0.7; with audio 0.6.*
- **Actual usage**: determined by the `usage` field in the API response.

# Image Generation Model (Seedream) Billing Description

The image generation model is billed by the **number of successfully generated images** (CNY/image).

## 1. Per-Image Pricing Table

| **Model Name**               | **Price (CNY/image)** |
| ---------------------------- | ---------------- |
| **doubao-seedream-5.0-lite** | 0.22             |
| **doubao-seedream-5.0**      | 0.22             |
| **doubao-seedream-4.5**      | 0.25             |
| **doubao-seedream-4.0**      | 0.20             |
| **doubao-seedream-3.0-t2i**  | 0.259            |

### Billing Rules:

- **Billing Condition**: Billed by the number of successfully output images. Images not successfully output due to review or other reasons are not charged.
- **Multiple image scenarios**: Billed by the actual number of generated images.

# Large Language Model (Text Generation) Billing Description

LLMs are billed by token on a pay-as-you-go basis; cost formula: `input price × input tokens + output price × output tokens`.

Some models use tiered billing based on input length; different price tiers apply to different ranges.
